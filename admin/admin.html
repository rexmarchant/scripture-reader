Workflow — All Phases
1 Generate SSML from scripture text Claude Project chat ▶
  • Save your scripture chapter as a .txt file.
  • Open the Scripture Reader Claude Project and start a new chat.
  • Upload the .txt file and say: convert this to SSML
  • Claude outputs two downloadable files:
    V1 SSML — verse numbers stripped, paste into Amazon Polly.
    V2 SSML — verse numbers retained, used by VTT Converter.
  • Save both files to the chapter's source folder.
Open Scripture Reader Project ↗
2a Generate audio with Amazon Polly AWS Console ▶
  • Go to AWS Console → Amazon Polly → Text to Speech.
  • Set Engine: Neural, Language: English (US), Voice: Patrick.
  • Switch input to SSML and paste the V1 SSML content.
  • Click Synthesize to S3. Choose your S3 bucket and click Synthesize.
  • Wait for the job to complete, then download the MP3 from S3.
  • Delete the file from S3 after downloading — avoid ongoing storage charges.
  • Save the MP3 to the chapter's source folder.
Open AWS Polly ↗
2b Generate verse-aligned VTT with Whisper Whisper CLI + vtt_converter.html ▶
  • Open a Command Prompt and run Whisper on the chapter MP3:
python -m whisper "path\to\chapter.mp3" --model small --output_format vtt
  • Whisper outputs a .vtt file in the same folder as the MP3 (or your current directory). Move it to the chapter's source folder.
  • Open VTT Converter and drop in the V2 SSML and the Whisper VTT.
  • Click Convert — review the confidence column in the preview table. High confidence = accurate timestamp. Low confidence rows may need spot-checking.
  • Click Download VTT to save the verse-aligned VTT file.
  • This verse-aligned VTT replaces the V2 SSML as input to the Audio Sync Player — exact timestamps, no manual tuning needed.
Open VTT Converter →
3 Upload MP3 to Archive.org archive.org ▶
  • Go to archive.org/upload and log in with your Google account.
  • Upload the MP3. Give it a descriptive title.
  • After upload, open the item page. In Download Options, right-click the VBR MP3 filename → Copy Link Address.
  • The URL must use /download/ not /details/:
    ✗ archive.org/details/your-item
    ✓ archive.org/download/your-item/file.mp3
  • Save the URL — you'll need it in Phase 4.
  • Play the file on Archive.org to confirm it's the correct voice and chapter.
Open Archive.org ↗
4 Sync and export chapter player audio_sync_player.html ▶
  • Open Audio Sync Player and drop in the MP3 and the verse-aligned VTT from Phase 2b.
  • The VTT timestamps are exact — no rate tuning needed. The blue info bar should show near-zero diff vs audio.
  • Play back a few spots to verify verse highlighting looks correct.
  • Fill in the Export panel: Scripture Set, Book, Chapter, MP3 URL (from Phase 3).
  • Click Export Chapter Player → two files download:
    book-chapter.html → move to chapterfiles/
    index.html → replace root index.html
  • Click Save JSON to file and save to the chapter source folder for reference.
  • Click Download export log and save locally (do not deploy).
Open Audio Sync Player →
5 Verify locally index.html ▶
  • Open your local index.html in a browser.
  • Confirm the new chapter appears in the library.
  • Open the chapter and verify audio plays and verse highlighting follows along correctly.
Open Local index.html →
6 Publish to Netlify netlify.com ▶
  • Copy the deploy files into your Scripture Reader publish folder (see Deploy Checklist below).
  • Log in to app.netlify.com with your Google account.
  • Open the bom-reader project.
  • Drag and drop the Scripture Reader folder onto the Production deploys box.
  • Wait for deploy to complete, then test at bom-reader.netlify.app.
Open Netlify ↗
