ssml_converter.html (new):
  - Drag-and-drop .txt scripture file
  - Calls Claude API to generate both SSML variants
  - Auto-downloads V1 SSML (verse numbers stripped — for Amazon Polly)
  - Auto-downloads V2 SSML (verse numbers retained — for VTT Converter)
  - Naming convention compliant filenames (e.g. 1-nephi-chapter-3-v1.ssml)

vtt_converter.html (new):
  - Accepts V2 SSML + Whisper VTT (drop both at once or one at a time)
  - Two-pass verse alignment engine:
      Pass 1: word-stream matching — flattens all VTT cues into a single
              word array with sub-cue interpolated timestamps; matches
              each verse's anchor words forward through the stream;
              pacing sanity check rejects implausible timestamp jumps
      Pass 2: interpolation — verses that couldn't be matched get
              timestamps distributed proportionally by character count
              between surrounding word-matched anchors
      Pass 3: deduplication — ensures every verse has a strictly later
              timestamp than the previous (minimum 0.1s gap)
  - Chapter title (e.g. "First Nephi, Chapter 3") extracted from SSML
    and output as a time:0 cue so the spoken announcement stays in sync
  - Verse 1 timestamp derived from its actual word position within the
    first Whisper cue (sub-cue interpolation), not the cue boundary
  - Conversion summary: verses aligned / word-matched / interpolated /
    low confidence counts
  - Preview table: every verse with timestamp, match type, and text
  - Downloads verse-aligned VTT ready for Audio Sync Player
  - No manual timing adjustment needed in Audio Sync Player when using
    a verse-aligned VTT

admin.html (new):
  - Sidebar navigation linking all tools and external services
  - Tool cards for quick launch
  - Expandable workflow phases (1 through 6) with inline launch buttons
  - Deploy file checklist (what to include vs exclude from Netlify)
  - External links: AWS Polly, AWS S3, Archive.org, Netlify, Live Site

audio_sync_player.html changes:
  - When a timed format (VTT/LRC/SRT) is loaded, shows an info bar
    confirming exact timestamps and time span
  - Timing Rate card dims and disables when a timed format is loaded
    (rate controls only apply to SSML/TXT character-based timing)
  - Rate controls restore to full opacity on Clear Files
  - No changes to export behavior or chapter HTML output

Whisper CLI (external tool — install once):
  python -m pip install openai-whisper
  winget install ffmpeg
  python -m whisper "chapter.mp3" --model small --output_format vtt

Old workflow (v4):
  .txt → SSML (Claude chat) → Polly MP3 →
  Audio Sync Player (manual timing tuning) → export

New workflow (v5):
  .txt → ssml_converter.html (V1 + V2 SSML) →
  Polly MP3 (V1 SSML) →
  Whisper CLI (raw VTT) →
  vtt_converter.html (verse-aligned VTT) →
  Audio Sync Player (no tuning needed) → export

Manual timing tuning step eliminated. Each verse now has a timestamp
derived from the actual Whisper transcription of the audio.
