  WORKFLOW — CREATING A CHAPTER
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Phase 1 — Generate SSML (ssml_converter.html)
  1. Open ssml_converter.html.
  2. Drag the .txt scripture file onto the drop zone.
  3. Two files auto-download:
       V1 SSML — verse numbers stripped, for Amazon Polly
       V2 SSML — verse numbers retained, for VTT Converter

Phase 2a — Generate audio (AWS Polly)
  1. Sign into AWS → Amazon Polly → Text to Speech.
  2. Engine: Neural, Language: English US, Voice: Patrick, SSML: ON.
  3. Paste V1 SSML content → Synthesize to S3 (bucket: rex-file-storage-2026).
  4. Download the MP3 from S3, then delete it from the bucket.

Phase 2b — Generate verse-aligned VTT (Whisper + vtt_converter.html)
  1. Run Whisper on the MP3 from a Command Prompt:
       python -m whisper "path\to\chapter.mp3" --model small --output_format vtt
  2. Open vtt_converter.html.
  3. Drop the V2 SSML and Whisper VTT onto the drop zone.
  4. Click Convert — review the summary (word-matched / interpolated / low).
  5. Click Download VTT — save the verse-aligned VTT to the source files folder.

Phase 3 — Upload MP3 to Archive.org
  1. Go to archive.org/upload and upload the MP3.
  2. After upload, right-click VBR MP3 in Download Options → Copy Link Address.
  3. Confirm the URL uses /download/ not /details/:
       WRONG:   https://archive.org/details/your-item
       CORRECT: https://archive.org/download/your-item/your-file.mp3

Phase 4 — Sync and export chapter (audio_sync_player.html)
  1. Open audio_sync_player.html.
  2. Drop the MP3 and verse-aligned VTT onto the drop zone.
     The blue info bar confirms exact VTT timestamps — no rate tuning needed.
  3. Play back a few spots to verify verse highlighting is in sync.
  4. Fill in the Export panel:
       Scripture Set → e.g. Book of Mormon
       Book          → e.g. 1 Nephi
       Chapter       → e.g. Chapter 3
       MP3 URL       → paste Archive.org /download/ URL from Phase 3
  5. Click "Export Chapter Player":
       book-of-mormon-1-nephi-chapter-3.html  → move to chapterfiles/
       index.html                              → replace root index.html
  6. Click "Save JSON to file" → save to source files folder.
  7. Click "Download export log" → save locally (do not deploy).

Phase 5 — Verify locally
  Open local index.html → confirm new chapter appears → open it and
  verify audio plays with verse highlighting in sync.

Phase 6 — Publish to Netlify
  1. Assemble the publish folder (see Folder Structure above).
  2. Login to app.netlify.com (Google account) → open bom-reader project.
  3. Drag and drop the Scripture Reader folder to Production deploys.
  4. Test at https://bom-reader.netlify.app/index.html.

  WORKFLOW — MANAGING THE LIBRARY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

scripture_reader.html is the admin library manager. Open it locally.

Adding chapters:
  - Paste the JSON from "Copy JSON" or "Save JSON to file" into the
    "Or paste exported chapter JSON" field, then click Add chapter.
  - Or fill in the manual fields (Scripture Set, Book, Chapter, file).

Removing chapters:
  - ✕ button on a chapter card — removes that chapter only
  - Remove book — removes one book and all its chapters
  - Remove set — removes an entire scripture set
  - Clear all — wipes the full library (with confirmation)
  All removals show a confirmation dialog before acting.

Regenerating index.html:
  - Click "Export index.html" in the Add chapter panel at any time
    to download a fresh public index from the current library state.

The library is stored in browser localStorage (key: scripture_reader_chapters_v2).
audio_sync_player.html and scripture_reader.html share the same library,
so chapters exported in one appear in the other automatically.

  EXPORT LOG
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every click of "Export Chapter Player" appends a record to localStorage.
Click "Download export log" to download the full log as scripture_export_log.csv.
Each download includes all records from the beginning — replace your
previous CSV with the new download to keep one up-to-date file.

CSV columns:
  Timestamp, Scripture Set, Book, Chapter, MP3 URL,
  Avg Chars Per Line, Seconds Used, Chars Per Unit, Audio File, Text File

Keep scripture_export_log.csv locally — do not deploy to Netlify.
Download it regularly since clearing browser data will clear the log.

  WHISPER SETUP (first time only)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Requirements: Python 3.x, pip, ffmpeg

Install once:
  python -m pip install openai-whisper
  winget install ffmpeg          (then reopen Command Prompt)

Run per chapter:
  python -m whisper "path\to\file.mp3" --model small --output_format vtt

Notes:
  - First run downloads the model (~460MB) — subsequent runs are fast
  - The FP16/FP32 warning on CPU is harmless
  - Output .vtt file goes to the same folder as the MP3 by default

  ARCHIVE.ORG TIPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- Always use the /download/ URL, never /details/
- Archive.org items are public by default — anyone can access the audio
- Files are permanent and free with no bandwidth limits
- If audio fails to load in the chapter player, the player will show
  the error type and the URL it tried to use for diagnosis

If you prefer private audio, place MP3s in a root audio/ folder on
Netlify and use a relative path like audio/filename.mp3 in the export
panel instead of an Archive.org URL.

  RESTORING A DELETED CHAPTER (v3 feature)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

If a chapter was accidentally removed from the library but the chapter
HTML file still exists in your chapterfiles/ folder and the audio is
still live on Archive.org, you can restore it in seconds:

  1. Open scripture_reader.html.
  2. Click "Add chapter" to expand the panel.
  3. At the top, click "Choose chapter HTML file".
  4. Select the .html file from your chapterfiles/ folder.
  5. The reader extracts Scripture Set, Book, Chapter, and audio URL
     automatically from the file and adds it back to the library.
  6. Click "Export index.html" to regenerate the public reader.
  7. Upload the new index.html to Netlify.

No re-timing, no re-export from audio_sync_player needed.
The chapter HTML file is read-only — it is not modified.
